# Domain Docs

This is a single-context repo.

## Before Exploring

Read root `CONTEXT.md` for domain language. Read ADRs under root `docs/adr/` when they exist and touch the area being changed.

If `docs/adr/` does not exist, proceed silently. ADRs are created lazily only when a hard-to-reverse, surprising, trade-off-heavy decision needs recording.

## Vocabulary

Use the terms defined in `CONTEXT.md` when writing issues, PRDs, refactor proposals, hypotheses, test names, or implementation notes.

If a needed concept is missing from `CONTEXT.md`, treat that as a prompt for `/grill-with-docs` rather than inventing new project terminology.
